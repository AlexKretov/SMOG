# SMOG: Synthetic Minority Over‑sampling with Generative models

[![Python 3.9+](https://img.shields.io/badge/python-3.9+-blue.svg)](https://www.python.org/downloads/)
[![License](https://img.shields.io/badge/license-Apache%202.0-green)](LICENSE)

**SMOG** — это исследовательский проект, демонстрирующий применение передовых генеративных моделей для синтеза наблюдений минорного класса в сильно несбалансированных табличных данных. Основная задача — улучшить качество моделей машинного обучения для обнаружения мошенничества и прогнозирования оттока клиентов на примере датасета мобильных денег в Нигерии.

В отличие от классических методов oversampling’а (SMOTE, ADASYN), SMOG использует **CTGAN, TabDDPM и гибридную фильтрацию**, что позволяет генерировать более реалистичные и разнообразные примеры редких классов, повышая обобщающую способность моделей.

---

## 📌 Цели проекта

1. **Сравнить качество** моделей классификации (мошенничество / отток) при обучении на:
   - исходных несбалансированных данных;
   - данных с oversampling’ом методами SMOTE / ADASYN;
   - данных, дополненных синтетическими примерами, сгенерированными CTGAN;
   - данных, дополненных синтетическими примерами, сгенерированными TabDDPM;
   - данных, отобранных через гибридную фильтрацию после генерации.
2. **Продемонстрировать** преимущества глубоких генеративных моделей перед интерполяционными методами.
3. **Оценить качество синтетических данных** с помощью визуализации (PCA, UMAP) и метрик (Discrimination Score, Coverage, Efficacy).

## 📊 Датасет

Используется датасет **Nigerian Mobile Money Wallet Transactions** с Hugging Face:

- **Ссылка:** [electricsheepafrica/nigerian-banking-mobile-money](https://huggingface.co/datasets/electricsheepafrica/nigerian-banking-mobile-money)
- **Объём:** 4 000 000 транзакций (в проекте используется пилотная версия 10 000 для быстрого прототипирования)
- **Целевые признаки:**
  - `fraud_flag` (мошенничество) — положительный класс **1.5%**
  - `churn_30d` (отток) — положительный класс **6%**
- **Признаки:** транзакционные, временные, клиентские (KYC, канал, устройство, баланс и др.)

Проведён **feature engineering**:
- циклическое кодирование времени (час, день недели, месяц);
- агрегации по кошелькам (количество транзакций, средняя сумма, доля cashout);
- бинарные флаги (ночная транзакция, круглые суммы, новый кошелёк и т.п.).

## 🧠 Методология

### 1. Базовые методы (Baseline)
- **SMOTE** (Synthetic Minority Over-sampling Technique)
- **ADASYN** (Adaptive Synthetic Sampling)

### 2. Глубокие генеративные модели
- **CTGAN** (Conditional Tabular GAN) — библиотека `sdv`
- **TabDDPM** (Tabular Denoising Diffusion Probabilistic Models) — адаптация диффузионной модели для табличных данных

### 3. Гибридный подход
- Генерация избыточного пула синтетических примеров минорного класса.
- Фильтрация с использованием базового классификатора (XGBoost) для отбора:
  - примеров с низкой уверенностью (близких к границе решения);
  - разнообразных примеров (по расстоянию в латентном пространстве).

### 4. Оценка качества синтетических данных
- **Discrimination Score** — способность модели отличать реальные данные от синтетических.
- **Coverage** — насколько синтетические примеры покрывают распределение реальных (метрика на основе kNN).
- **Efficacy** — улучшение F1-меры и PR-AUC на тестовой выборке.
